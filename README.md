## Адаптивная торговая система на основе рыночных режимов

Данный проект реализует **адаптивную торговую систему для рынка форекс**, которая автоматически:

- **детектирует рыночный режим** (тренд / флет / высокая волатильность),
- **подбирает стратегию** под режим (mean reversion или momentum breakout),
- **управляет риском** через динамический размер позиции, адаптивный стоп-лосс и фильтрацию по корреляции валютных пар.

Система спроектирована модульно и пригодна как для **бэктестинга**, так и для дальнейшей интеграции с реальным исполнением у брокера.

---

## 1. Общая концепция

Высокоэффективная стратегия на форекс должна **адаптироваться к типу рынка**, а не пытаться одной логикой торговать всё подряд.

В основе системы лежит трёхэтапный конвейер:

1. **Режимная детекция (Market Regime)**
2. **Выбор стратегии (Strategy Selection)**
3. **Исполнение + риск-менеджмент (Execution & Risk Management)**

Текстовая схема:

1. **РЕЖИМНАЯ ДЕТЕКЦИЯ**
   - **ADX** — измеряет силу тренда.
   - **Bollinger Bands Width (BB Width)** — оценивает сжатие/расширение волатильности.
   - **ATR** — динамика диапазона цены (истинный диапазон).
   - **Price vs Moving Average (MA)** — положение цены относительно скользящей средней (центр / край канала).

2. **ВЫБОР СТРАТЕГИИ**
   - Если **ADX \< 20** и **BB сужается** → режим **Mean Reversion (возврат к среднему)**.
   - Если **ADX \> 25** и происходит **пробой уровня / диапазона** → режим **Momentum Breakout (торговля по импульсу)**.
   - Иначе → **режим ожидания** или **снижение торгового объёма**.

3. **ИСПОЛНЕНИЕ + РИСК-МЕНЕДЖМЕНТ**
   - **Динамический размер позиции** на основе волатильности и допустимого риска по счёту.
   - **Адаптивный стоп-лосс** на базе ATR и структуры цены.
   - **Фильтр корреляции пар**, чтобы не накапливать однонаправленный риск на сильно коррелирующих инструментах.

## 2. Архитектура проекта

Планируемая структура кода (минимальная и расширяемая):

- `indicators.py` — вычисление индикаторов:
  - ADX,
  - ширина полос Боллинджера,
  - ATR,
  - скользящие средние.
- `regime_detection.py` — логика определения рыночного режима на основе индикаторов.
- `strategy_selection.py` — выбор активной стратегии (mean reversion / momentum breakout / ожидание).
- `risk_management.py` — размер позиции, стоп-лосс, фильтр корреляции валютных пар.
- `engine.py` — «движок» системы, объединяющий модули, принимающий исторические данные и выдающий торговые сигналы.
- `tests/` — автотесты для ключевых компонентов (индикаторы, режимы, выбор стратегии, риск).

В дальнейшем эти модули будут реализованы по принципу **минимализма и расширяемости**:

- чётко определённые интерфейсы функций и классов,
- отсутствие жёсткой привязки к конкретному брокеру,
- возможность заменить часть логики (например, другую модель детекции тренда) без переписывания всего кода.

## 3. Логика режимной детекции (Market Regime)

**Цель:** на каждом баре (свечке) определить состояние рынка:

- тренд (вверх / вниз),
- флет (диапазонная торговля),
- переходные/неопределённые состояния.

### 3.1. Используемые метрики

- **ADX (Average Directional Index)**:
  - **низкий ADX (\< 20)** → слабый тренд / боковик;
  - **средний ADX (20–25)** → переходная зона;
  - **высокий ADX (\> 25)** → выраженный тренд.
- **Bollinger Bands Width (ширина полос)**:
  - нормируется относительно средней цены;
  - **сужение ширины** → снижение волатильности, подготовка к возможному импульсу;
  - **расширение ширины** → всплеск волатильности, часто во время тренда или его начала.
- **ATR (Average True Range)**:
  - абсолютная волатильность,
  - используется как для оценки режима, так и для расчёта стоп-лоссов и размера позиции.
- **Price vs MA**:
  - цена около MA → срединная зона, характерна для mean reversion;
  - цена далеко от MA → возможное продолжение/перекупленность тренда.

### 3.2. Правила классификации режима (базовая версия)

- **Режим Mean Reversion**:
  - ADX \< 20,
  - BB Width падает относительно скользящего среднего значения своей истории,
  - цена колеблется вокруг средней (MA) без устойчивого тренда.

- **Режим Momentum Breakout**:
  - ADX \> 25,
  - BB Width расширяется,
  - цена пробивает локальный экстремум или границу диапазона (по простому high/low за N баров).

- **Режим ожидания / неопределённости**:
  - ADX в промежуточной зоне (20–25),
  - индикаторы дают противоречивые сигналы.

Пороговые значения (20 и 25) и окна индикаторов будут вынесены в **конфигурацию**, чтобы их можно было подбирать в бэктестинге.

## 4. Стратегии

### 4.1. Mean Reversion

**Идея:** когда рынок во флете и волатильность сжата, выгодно торговать от границ диапазона к средней.

Варианты сигналов:

- вход в **лонг**, если:
  - цена опустилась к нижней полосе Боллинджера или ниже неё,
  - режим = Mean Reversion,
  - нет сильного трендового сигнала по ADX.
- вход в **шорт**, если:
  - цена поднялась к верхней полосе Боллинджера или выше её,
  - режим = Mean Reversion.

Выход:

- тейк-профит возле средней (MA),
- стоп-лосс за пределами диапазона (на базе ATR или за границей предыдущего экстремума).

### 4.2. Momentum Breakout

**Идея:** при наличии тренда и всплеска волатильности выгодно заходить по направлению импульса после пробоя.

Сигналы:

- вход в **лонг**, если:
  - ADX \> 25,
  - цена пробивает максимум за N баров,
  - BB Width расширяется,
  - режим = Momentum Breakout.
- вход в **шорт**, если:
  - ADX \> 25,
  - цена пробивает минимум за N баров,
  - BB Width расширяется.

Выход:

- трейлинг-стоп по ATR,
- частичная фиксация по фиксированному RR (например, 1:2) с переводом остатка в безубыток.

## 5. Риск-менеджмент

### 5.1. Динамический размер позиции

Базовый подход:

- задаётся **максимальный риск на сделку** в процентах от капитала (например, 1%),
- стоп-лосс измеряется в пунктах через ATR или структуру экстремумов,
- размер позиции рассчитывается так, чтобы при срабатывании стопа **потеря была не больше заданного процента**.

Формула:

\[
position\_size = \frac{account\_equity \times risk\_per\_trade}{stop\_distance\_in\_price \times value\_per\_point}
\]

где:

- `account_equity` — текущий капитал,
- `risk_per_trade` — доля риска (0.01 для 1%),
- `stop_distance_in_price` — расстояние от входа до стопа,
- `value_per_point` — стоимость пункта для конкретной пары/лотности.

### 5.2. Адаптивный стоп-лосс

- в спокойном рынке (низкий ATR) стоп может быть **ближе**,
- в высоковолатильном рынке (высокий ATR) стоп **расширяется**, чтобы не выносило шумом.

Базовая идея:

\[
stop\_distance = k \times ATR
\]

где `k` зависит от стратегии и режима (например, 1.5–3 ATR).

### 5.3. Фильтр корреляции пар

**Цель:** не открывать чрезмерно схожие позиции по сильно коррелирующим парам (например, EURUSD и GBPUSD в один и тот же момент в одном направлении).

Подход:

- на основе исторических доходностей рассчитывается **скользящая корреляция** между парами,
- при принятии решения об открытии новой позиции проверяется:
  - не превышает ли суммарная экспозиция по высоко коррелированным парам заданный предел,
  - при необходимости размер новой позиции снижается или сделка блокируется.

Проект ориентирован на **Python 3.10+**.

Минимальные внешние зависимости:

- `numpy` — численные расчёты,

1. Создать виртуальное окружение:
   - Windows (PowerShell): `python -m venv .venv`

   - Windows (PowerShell): `.venv\\Scripts\\Activate.ps1`
3. Установить зависимости (будут также перечислены в `requirements.txt`):
   - `pip install -U pip`
   - `pip install numpy pandas`

## 7. Тестирование

Для проверки корректности логики будут реализованы **автотесты** (на базе стандартного модуля `unittest`), покрывающие:

- вычисление индикаторов (ADX, ATR, Bollinger Bands Width, MA),
- классификацию рыночных режимов,
- выбор стратегии по набору индикаторов,
- расчёт размера позиции и стоп-лосса,
- работу фильтра корреляции (на синтетических данных).

Запуск тестов (после реализации кода):

- `python -m unittest discover`

## 8. Дальнейшее развитие

После базовой реализации ядра системы возможны следующие расширения:

- добавление более продвинутых фильтров тренда (например, регрессия, фильтр Хёрста, спектральный анализ),

- **минимально необходимый, но рабочий код**,
